Initialisation d’une nouvelle application React + Spring boot

L'objectif principal de cette réalisation est de démarrer et d'initialiser le développement d'une nouvelle application SaaS interne à Bouygues Telecom.
Destinée à deux équipes, cette application permettra de gérer efficacement les terminaux. Bien que je n'aie pas choisi ce projet, il s'inscrit dans la continuité de mon travail au sein d'une équipe de développement, où nous avons précédemment travaillé sur une application de gestion d'offres.
Le succès de cette dernière a conduit à la création d'une application similaire, mais avec des données différentes.

Contexte de la réalisation

Bouygues Telecom est un opérateur de télécommunications majeur en France. En tant qu'alternant, je suis développeur full-stack au sein de la Direction des Systèmes d'Information (DSI).
Le besoin de créer cette nouvelle application est motivé par la nécessité de gérer plus facilement et efficacement l'ensemble des terminaux.

Présentation et définition du projet

L'application développée a pour fonction principale de faciliter la gestion des terminaux.
Le front-end est développé en React TypeScript, en utilisant des librairies internes de Bouygues Telecom, notamment leur librairie de composants graphiques.
Le back-end est conçu en Java Spring Boot, avec une base de données PostgreSQL. L'accent est mis sur l'initialisation de l'application plutôt que sur ses fonctionnalités spécifiques.

Enjeux

Le principal défi de ce projet réside dans le timing serré : seulement 12 jours-homme pour une équipe de deux développeurs, dont moi-même, pour initialiser l'application.
À la fin de cette période, l'application devait être déployée sur un environnement de développement, avec une CI/CD mise en place et une base de données initialisée. Les technologies utilisées sont celles préconisées en interne, ce qui a influencé nos choix techniques et méthodologiques.

Utilisation des compétences acquises

J'ai mis en œuvre mes compétences en développement front-end avec React, en installant les librairies nécessaires et en utilisant la librairie de composants graphiques de Bouygues Telecom.
J'ai également utilisé mes connaissances en CI/CD pour écrire des scripts YAML et implémenté la navbar de l'application avec l'aide de Github Copilot.
Ce projet m'a permis de renforcer mes compétences en TypeScript, React, CI/CD, YAML, Java, et Spring Boot.

Planification et étapes de réalisation

Les grandes étapes de la réalisation du projet incluent :

  • La création d'un repository GitLab
  • Le développement d'une application React "Hello World" et son déploiement sur un environnement de développement
  • La même démarche pour le back-end avec une application Spring Boot vierge
  • La mise en communication du front-end et du back-end via une API simple
  • Le développement de la première page d'accueil avec un menu navbar et une authentification

La planification et le suivi des tâches ont été réalisés dans le cadre d'un sprint court, avec chaque étape notée dans une user story sur Jira.

Regard critique sur le résultat

À ce jour, j'ai réussi à avancer dans les délais impartis et j'ai l'impression d'avoir écrit un code propre pour l'architecture de l'application front-end.
Cependant, j'aurais aimé implémenter l'architecture hexagonale dans le back-end, mais le manque de temps ne l'a pas permis. La finalisation de ce projet dépendra de ma situation professionnelle future, car je termine mon alternance en cours de projet.

Conclusion

Cette expérience m'a appris que l'initialisation d'un nouveau projet prend toujours plus de temps que prévu, surtout dans une grande entreprise où de nombreux outils internes nécessitent des configurations spécifiques.
Ce projet enrichit mon parcours professionnel en tant que développeur full-stack et me servira de référence pour mes futurs projets, que ce soit au sein de Bouygues Telecom ou dans mes emplois futurs.
